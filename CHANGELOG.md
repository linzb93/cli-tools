# 更新日志

## 7.3.0

### 功能

-   优化`open`方法，解决某些终端无法结束任务的bug。
-   `git commit`命令新增`merge`选项，支持与上一条记录合并。
-   Jenkins配置移除`lastTag`字段，暂时不需要。
-   `iteration`命令替换成`git iteration`命令。支持公司项目和Github项目的普通结构和Monorepo结构。
-   新增`npm scan`命令，用于扫描是否有不安全的npm包。
-   新增`minimax`命令，用于查看Minimax API的使用情况。

## 7.2.0 2026-03-25

### 功能

-   `version`命令替换成`iteration`命令，用于版本迭代：更新版本号并处理Git工作流。
-   移除`beauty`命令和`lixi`命令，`ai ocr`命令提升为`ocr`命令，移除`ai regex`命令。
-   token解析不了的，打开指定网站解析。
-   vscode插件支持复制选中代码行号。
-   Jenkins配置新增`lastTag`字段，用于记录上一个版本的标签，避免线上tag被清空后还要重头创建版本tag。
-   新增`cd`命令，方便进入最近的项目目录。

### 代码重构

-   开发和正式打包文件放在不同目录下。
-   所有命令的回调改成动态引入，提高运行速度。
-   `shared`项目里面所有文件都可以通过`index.js`导出。
-   git clone、pull、push命令会因为网络问题失败重试的，统一管理错误信息。

### bug修复

-   修复git三级命令无法使用的bug。

## 7.1.0 2026-03-18

### 功能

-   支持更多 commit 类型。
-   优化 git 提交时的关键字判断逻辑。
-   version命令在创建分支前会判断分支是否存在。

### 代码重构

-   优化 GitHub 推送流程：优先推送，按需拉取合并。
-   优化 awesome 命令交互体验。
-   优化 git push：允许失败后重试提交。

## 7.0.0 2026-03-06

### 功能

-   不再支持 NodeJS 18 以下的版本，npm依赖全部升级。
-   `git scan`功能支持在终端不用打开项目目录，直接提交和推送代码。
-   `awesome`命令交互由终端改成Web前端，减少终端代码。
-   用python编写Windows右键菜单图像处理的代码。

### bug修复

-   修复`git pull`和`git push`命令在非超时报错时，会重复调用的bug。

### 代码重构

-   接口 router 回调函数由 Promise 改成使用 async/await。

## 6.0.0 2026-02-17

### 功能

-   tree 命令：新增选项`comment`，含义是添加注释。
-   git scan 命令：支持查看已提交未推送项目的提交详情以及文件。
-   occ 命令添加涨单宝。
-   新增 version 命令，新项目一键创建分支、修改版本等。
-   新增 awesome 命令，用于管理收集的 npm 包，支持增删改查。

### bug 修复

-   git pull 功能修复需要合并时，因为 git 钩子限制，无法提交的 bug。

### 代码重构

-   规范目录结构。
-   多态统一使用工厂模式。
-   业务代码不再使用面向对象，改成函数式，便于测试以及缩小打包大小。
