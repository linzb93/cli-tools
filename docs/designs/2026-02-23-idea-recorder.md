# 功能: 功能想法记录器 (Idea Recorder)

**日期**: 2026-02-23
**状态**: 计划中
**相关命令**: `idea` (新命令)
**业务代码位置**: `packages/cli/src/business/idea`

## 背景 / Context

开发者在开发其他项目时，经常会突发奇想，有一些关于本工具 (`cli-tools`) 的新功能点子。需要一个快速的命令将这些想法记录下来，以便后续有空时统一查阅和实现。

## 需求

-   **核心目标**: 快速记录，统一查阅。
-   **输入方式**:
    -   命令行参数: `idea add "我的想法"`
    -   剪贴板读取: 如果未提供参数，自动读取系统剪贴板内容。
-   **存储位置**: 本项目 (`cli-tools`) 根目录下的 `docs/ideas.md` 文件。
-   **查阅方式**:
    -   直接查看 `docs/ideas.md` 文件。
    -   (可选) 提供 `idea list` 命令查看。
-   **多条记录**: 新的想法追加到文件末尾，不覆盖现有内容。

## 方案对比

-   **方案 A: Markdown 追加模式 (推荐)**
    -   **描述**: 将每条想法格式化为 Markdown 列表项，追加到 `docs/ideas.md`。
    -   **格式**: `- [ ] [YYYY-MM-DD HH:mm] 想法内容`
    -   **优点**:
        -   人类可读性极佳，直接用编辑器或 GitHub 查看都很方便。
        -   自带 "Todo" 属性 (`[ ]`)，方便后续勾选完成。
        -   实现简单。
    -   **缺点**: 结构化查询稍弱（但对于记录想法足矣）。

-   **方案 B: JSON 数据库**
    -   **描述**: 存储在 JSON 文件中，通过 CLI 命令查看。
    -   **优点**: 结构化好，方便后续扩展（如标签、优先级）。
    -   **缺点**: 直接查看文件不直观，必须依赖 CLI 工具。

## 最终实现方案

-   **选定方案**: 方案 A (Markdown 追加模式)
-   **CLI 接口设计**:
    -   `idea add [content]`: 添加想法。
        -   如果 `[content]` 存在，直接记录。
        -   如果 `[content]` 为空，读取剪贴板内容记录。
    -   `idea list` (可选): 打印 `docs/ideas.md` 内容到控制台。
-   **存储格式**:
    ```markdown
    # Ideas

    - [ ] [2026-02-23 10:00] 支持自动生成 API 文档
    - [ ] [2026-02-23 11:30] 优化 build 速度
    ```
-   **行为说明**:
    -   命令执行时，检查 `docs/ideas.md` 是否存在，不存在则创建（包含标题）。
    -   获取当前时间戳。
    -   将内容追加到文件末尾。
    -   成功后提示用户 "Idea recorded: <content>..."。
