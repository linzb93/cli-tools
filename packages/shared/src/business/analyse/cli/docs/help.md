# CLI Usage Analysis

统计和分析 CLI 工具的使用情况，展示各命令和子命令的调用频率。

## 功能特性

-   **多维度统计**：支持按今日、本周、本月或全部时间进行数据分析。
-   **智能聚合**：自动识别并聚合主命令（如 `git`, `npm`）及其子命令的统计。
-   **排序展示**：按使用频率从高到低排序，直观展示高频命令。
-   **层级视图**：清晰展示命令与子命令的层级关系及各自的使用次数。

## 使用方法

```bash
$ cli-tools analyse cli [options]
```

## 选项说明

### --period

指定统计的时间周期。

-   **类型**：`string`
-   **可选值**：
    -   `day`: 统计今日的数据（限制在最近一个月内）
    -   `week`: 统计本周的数据（限制在最近一个月内）
    -   `month`: 统计本月的数据
    -   `all`: 统计所有历史数据
-   **默认值**：`all`

## 数据来源与过滤

-   数据来源于 `cacheRoot/track` 目录下的日志文件。
-   `day` 和 `week` 模式下，会自动过滤掉一个月前的数据，以保证查询效率和相关性。

## 注意事项

-   确保 CLI 工具已正确配置并开启了日志追踪功能，否则无数据可供分析。
-   日志文件按文件名排序读取，分析结果依赖于日志文件的时间戳准确性。
