# Git 仓库扫描

批量扫描本地 Git 仓库状态，快速检查多个仓库的提交、推送和分支状态。

## 功能特性

-   批量扫描多个 Git 仓库的状态
-   检测未提交的变更
-   检测未推送的提交
-   显示当前分支信息
-   提供详细的未推送提交日志
-   支持进度条显示扫描进度

## 使用方法

```bash
$ mycli git scan [options]
```

### 示例

```bash
# 基础扫描（排除正常状态的仓库）
$ mycli git scan

# 全量扫描（显示所有仓库状态）
$ mycli git scan --full
```

## 选项说明

### --full

是否进行全量扫描，显示所有仓库状态（包括正常状态的仓库）。

类型：`boolean`

默认值：`false`

## 状态说明

扫描结果中的状态标识：

-   🔴 **未提交**：存在未提交的变更
-   🟡 **未推送**：存在未推送的提交
-   🟢 **正常**：仓库状态正常
-   ⚪ **不在主分支上**：当前不在主分支上

##
 输出格式

扫描结果以表格形式展示，包含以下信息：

| 名称 | 地址 | 状态 | 分支 |
|------|------|------|------|
| 仓库名称 | 完整路径 | 状态标识 | 当前分支 |

对于未推送的仓库，还会显示详细的提交日志，包括：
- 提交信息
- 提交日期
- 修改的文件列表

## 使用场景

- 批量检查多个项目的 Git 仓库状态
- 快速定位需要提交、推送或切换分支的仓库
- 日常开发工作流中的仓库状态检查

## 注意事项

- 需要在包含多个 Git 仓库的父目录中运行
- 扫描过程会显示进度条
- 未推送详情显示最近3条提交记录（可配置）
- 自动忽略正常状态的仓库（除非使用 `--full` 选项）
